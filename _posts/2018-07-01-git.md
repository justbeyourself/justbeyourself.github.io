## 仓库初始化与管理

### 在本地初始化新仓库
```bash
git init
```

### 将文件纳入版本控制
```bash
git add .
```

### 提交
```bash
git commit -m "commit message"
```

### 检查当前文件状态
```bash
git status
```

## 远程仓库操作

### 从现有库克隆
```bash
git clone https://github.com/…/PrettyGirls.git 
```

### 查看远程仓库地址
```bash
git remote -v
```

### 更换远程仓库地址
```bash
git remote set-url origin URL 
```

### 删除现有远程仓库
```bash
git remote rm origin 
```

### 添加新远程仓库
```bash
git remote add origin url 
```

### 查看分支
```bash
# 查看本地和远程所有分支
git branch -a
```

### 检出远程分支
```bash
# 在本地起名为 dev 分支，并切换到本地的 dev 分支
git checkout -b dev origin/dev 
```

## 撤销与回退

### 撤销 git commit 操作
```bash
# 撤销 commit，保留修改
git reset --soft <commit_id>  
```

### 回退到某个 commit
```bash
# 回退到某个 commit，不保留修改 (<commit_id> 从 git log 中取，取前 7 位即可)
git reset --hard <commit_id> 
```

### 撤销 git push 操作
```bash
# 撤销已 push 的 commit
git revert <commit_id>
```
> **原理**: `git revert` 会产生一个新的 commit，它和指定 SHA 对应的 commit 是相反的。任何从原先的 commit 里删除的内容会在新的 commit 里被加回去，任何在原先的 commit 里加入的内容会在新的 commit 里被删除。这是 Git 最安全、最基本的撤销场景，因为它并不会改变历史。


## 设置代理

```
设置代理：

git config --global http.proxy 'socks5://127.0.0.1:5432' 
git config --global https.proxy 'socks5://127.0.0.1:1080'
 
#只对github.com
git config --global http.https://github.com.proxy socks5://127.0.0.1:1080

查看代理：

git config --global --get http.proxy
git config --global --get https.proxy
 

取消代理：

git config --global --unset http.proxy
git config --global --unset https.proxy
```

## 一个项目设置两个git地址，并最终实现一次性同时推送到两个git地址

```
git remote set-url --add origin https://git.growingio.cn/growing/backend/nuza-cdp.git   //给origin添加一个远程push地址，这样一次push就能同时push到两个地址上面
git remote -v //查看是否多了一条push地址（这个可不执行）

删除
git remote set-url --delete origin git@gitee.com:teamemory/myH5.git
```
